# TG WS Proxy (Android / Termux)

Локальный SOCKS5-прокси для Telegram на Android, запускаемый через Termux. Перенаправляет трафик через WebSocket-соединения к серверам Telegram, помогая обойти блокировки провайдера и ускорить работу приложения.

**Эффект аналогичен прокидыванию hosts для Web Telegram**: ускорение загрузки файлов, сообщений, обход ограничений.

## Как это работает

```
Telegram Android → SOCKS5 (127.0.0.1:1080) → TG WS Proxy → WSS (kws*.web.telegram.org) → Telegram DC
```

1. Поднимает локальный SOCKS5-прокси на `127.0.0.1:1080`.
2. Перехватывает подключения к IP-адресам Telegram.
3. Извлекает DC ID из MTProto obfuscation init-пакета.
4. Устанавливает WebSocket (TLS) соединение к DC через `kws{N}.web.telegram.org`.
5. Если WS недоступен — автоматически переключается на прямое TCP.

## Установка

Установите [Termux](https://f-droid.org/packages/com.termux/) (только F-Droid, не Google Play) и выполните одну команду:

```bash
curl -fsSL https://raw.githubusercontent.com/SurgeDriver/tg-ws-proxy-android/main/install.sh | bash
```

Скрипт установит зависимости, клонирует репозиторий, создаст конфиг и добавит команду `tgproxy` — сразу, без перезапуска сессии.

## Запуск

```bash
tgproxy
```

При каждом новом запуске Termux просто выполняй `tgproxy`.

## Настройка Telegram

1. **Настройки** → **Данные и память** → **Настройки прокси** → **Добавить прокси**
2. Тип: **SOCKS5**, сервер: `127.0.0.1`, порт: `1080`, логин/пароль — пусто
3. Сохранить и активировать

## Конфигурация

Файл: `~/TgWsProxy/config.json`

```json
{
  "port": 1080,
  "host": "127.0.0.1",
  "dc_ip": [
    "2:149.154.167.220",
    "3:149.154.175.100",
    "4:149.154.167.220",
    "5:91.108.56.190"
  ],
  "verbose": false
}
```

| Параметр | Описание |
|---|---|
| `port` | Порт SOCKS5-прокси (по умолчанию 1080) |
| `host` | Адрес прослушивания (по умолчанию 127.0.0.1) |
| `dc_ip` | Список DC в формате `DC:IP` |
| `verbose` | `true` для подробного логирования |

**Рекомендуемые DC по региону:**

| DC | Локация | IP |
|---|---|---|
| DC2 | Амстердам | `149.154.167.51` |
| DC3 | Майами | `149.154.175.100` |
| DC5 | Сингапур | `91.108.56.190` |

Для России рекомендуется DC2.

## Решение проблем

- **`[Errno 98] address already in use`** — порт занят старым процессом: `pkill -f android.py && tgproxy`
- **Прокси отваливается** — Android усыпляет процессы. `install.sh` вызывает `termux-wake-lock` автоматически; при ручном запуске выполни его сам перед `tgproxy`
- **Ошибки при установке** — убедись, что Termux из F-Droid, а не Google Play
- **Высокий пинг** — попробуй другой DC в `config.json`
