# TG WS Proxy (Android / Termux)

Локальный SOCKS5-прокси для Telegram на Android, запускаемый через Termux. Перенаправляет трафик через WebSocket-соединения к серверам Telegram, помогая обойти блокировки провайдера и ускорить работу приложения.

**Эффект аналогичен прокидыванию hosts для Web Telegram**: ускорение загрузки файлов, сообщений, обход ограничений.

## Как это работает

```
Telegram Android → SOCKS5 (127.0.0.1:<port>) → TG WS Proxy → WSS (kws*.web.telegram.org) → Telegram DC
```

1. Поднимает локальный SOCKS5-прокси на `127.0.0.1` с постоянными случайными учётными данными.
2. Порт, логин и пароль генерируются один раз при установке (`install.sh`) и сохраняются в `config.json`.
3. Перехватывает подключения к IP-адресам Telegram.
4. Извлекает DC ID из MTProto obfuscation init-пакета.
5. Устанавливает WebSocket (TLS) соединение к DC через `kws{N}.web.telegram.org`.
6. Если WS недоступен — автоматически переключается на прямое TCP.

## Установка

Установите [Termux](https://f-droid.org/packages/com.termux/) (только F-Droid, не Google Play) и выполните одну команду:

```bash
curl -fsSL https://raw.githubusercontent.com/SurgeDriver/tg-ws-proxy-android/main/install.sh | bash
```

Скрипт установит зависимости, сгенерирует случайные порт и учётные данные, запишет их в `config.json` и покажет готовые настройки для Telegram:

```
=====================================
 Telegram proxy settings (save these)
=====================================
 Host:     127.0.0.1
 Port:     47293
 Username: k9m2x7q4n1p8r3t6
 Password: w5c0z8n3q2x7m4k1r9p6t3v0
=====================================
```

**Запиши их один раз в Telegram — больше они не изменятся.**

## Запуск

```bash
tgproxy
```

При каждом новом запуске Termux просто выполняй `tgproxy`.

## Настройка Telegram

1. **Настройки** → **Данные и память** → **Настройки прокси** → **Добавить прокси**
2. Тип: **SOCKS5**, сервер: `127.0.0.1`, порт: **из строки выше**
3. Логин и пароль: **из строки выше**
4. Сохранить и активировать

> Настройки не меняются между запусками. Если `config.json` удалён — переустанови через `install.sh`.

## Конфигурация

Файл: `~/TgWsProxy/config.json`

```json
{
  "port": 47293,
  "host": "127.0.0.1",
  "dc_ip": [
    "2:149.154.167.220",
    "3:149.154.175.100",
    "4:149.154.167.220",
    "5:91.108.56.190"
  ],
  "verbose": false,
  "username": "k9m2x7q4n1p8r3t6",
  "password": "w5c0z8n3q2x7m4k1r9p6t3v0",
  "no_auth": false
}
```

| Параметр | Описание |
|---|---|
| `port` | Порт SOCKS5-прокси. Генерируется случайно (32768–60999) при установке |
| `host` | Адрес прослушивания (127.0.0.1) |
| `dc_ip` | Список DC в формате `DC:IP` |
| `verbose` | `true` для подробного логирования |
| `username` | Логин SOCKS5. Генерируется случайно при установке |
| `password` | Пароль SOCKS5. Генерируется случайно при установке |
| `no_auth` | `true` отключает аутентификацию полностью (не рекомендуется) |

**Рекомендуемые DC по региону:**

| DC | Локация | IP |
|---|---|---|
| DC2 | Амстердам | `149.154.167.51` |
| DC3 | Майами | `149.154.175.100` |
| DC5 | Сингапур | `91.108.56.190` |

Для России рекомендуется DC2.

## Безопасность

- Прокси слушает **только** на `127.0.0.1` — недоступен из сети.
- Порт, логин и пароль уникальны для каждой установки и не публикуются.
- Клиент без корректных credentials получает SOCKS5 rejection и отключается.
- Для максимальной изоляции не устанавливай недоверенные приложения на то же устройство.

## Решение проблем

- **`[Errno 98] address already in use`** — порт занят старым процессом: `pkill -f android.py && tgproxy`
- **Прокси отваливается** — Android усыпляет процессы. `install.sh` вызывает `termux-wake-lock` автоматически; при ручном запуске выполни его сам перед `tgproxy`
- **Ошибки при установке** — убедись, что Termux из F-Droid, а не Google Play
- **Высокий пинг** — попробуй другой DC в `config.json`
- **Telegram не подключается после переустановки** — порт и credentials обновились, обнови их в настройках прокси Telegram
