# TG WS Proxy (Android / Termux)

Локальный SOCKS5-прокси для Telegram на Android, запускаемый через Termux. Перенаправляет трафик через WebSocket-соединения к серверам Telegram, помогая обойти блокировки провайдера и ускорить работу приложения.

**Эффект аналогичен прокидыванию hosts для Web Telegram**: ускорение загрузки файлов, сообщений, обход ограничений.

## Как это работает

```
Telegram Android → SOCKS5 (127.0.0.1:<random_port>) → TG WS Proxy → WSS (kws*.web.telegram.org) → Telegram DC
```

1. Поднимает локальный SOCKS5-прокси на `127.0.0.1` со **случайным портом** (при каждом запуске новый).
2. При каждом запуске генерируются **случайные логин и пароль** для SOCKS5 — незапрошенное подключение получает отказ.
3. Перехватывает подключения к IP-адресам Telegram.
4. Извлекает DC ID из MTProto obfuscation init-пакета.
5. Устанавливает WebSocket (TLS) соединение к DC через `kws{N}.web.telegram.org`.
6. Если WS недоступен — автоматически переключается на прямое TCP.

## Установка

Установите [Termux](https://f-droid.org/packages/com.termux/) (только F-Droid, не Google Play) и выполните одну команду:

```bash
curl -fsSL https://raw.githubusercontent.com/SurgeDriver/tg-ws-proxy-android/main/install.sh | bash
```

Скрипт установит зависимости, клонирует репозиторий, создаст конфиг и добавит команду `tgproxy` — сразу, без перезапуска сессии.

## Запуск

```bash
tgproxy
```

При каждом запуске в терминале появится строка вида:

```
Proxy ready on 127.0.0.1:54321  [DC2:..., DC5:...]
SOCKS5 credentials  user=a3f7c2e1  pass=9b4d0f82
Telegram proxy string: socks5://a3f7c2e1:9b4d0f82@127.0.0.1:54321
```

Скопируй эти данные и введи их в настройках Telegram.

## Настройка Telegram

1. **Настройки** → **Данные и память** → **Настройки прокси** → **Добавить прокси**
2. Тип: **SOCKS5**, сервер: `127.0.0.1`, порт: **из строки выше**
3. Логин и пароль: **из строки выше**
4. Сохранить и активировать

> Порт и учётные данные меняются при каждом перезапуске `tgproxy`. После рестарта нужно обновить их в настройках Telegram.

## Конфигурация

Файл: `~/TgWsProxy/config.json`

```json
{
  "port": 0,
  "host": "127.0.0.1",
  "dc_ip": [
    "2:149.154.167.220",
    "3:149.154.175.100",
    "4:149.154.167.220",
    "5:91.108.56.190"
  ],
  "verbose": false,
  "username": "",
  "password": "",
  "no_auth": false
}
```

| Параметр | Описание |
|---|---|
| `port` | Порт SOCKS5-прокси. `0` = случайный свободный порт (рекомендуется) |
| `host` | Адрес прослушивания (по умолчанию 127.0.0.1) |
| `dc_ip` | Список DC в формате `DC:IP` |
| `verbose` | `true` для подробного логирования |
| `username` | Фиксированный логин SOCKS5. Пусто = генерировать случайно |
| `password` | Фиксированный пароль SOCKS5. Пусто = генерировать случайно |
| `no_auth` | `true` отключает аутентификацию полностью (не рекомендуется) |

**Рекомендуемые DC по региону:**

| DC | Локация | IP |
|---|---|---|
| DC2 | Амстердам | `149.154.167.51` |
| DC3 | Майами | `149.154.175.100` |
| DC5 | Сингапур | `91.108.56.190` |

Для России рекомендуется DC2.

## Безопасность

- Прокси слушает **только** на `127.0.0.1` — недоступен из сети.
- При каждом запуске генерируются новые случайные учётные данные, исключая использование прокси сторонними приложениями.
- Клиент, не предоставивший корректный логин/пароль, получает ошибку SOCKS5 и отключается.
- Для максимальной изоляции не устанавливайте недоверенные приложения на то же устройство.

## Решение проблем

- **`[Errno 98] address already in use`** — при `port: 0` этого не случится. Если используется фиксированный порт: `pkill -f android.py && tgproxy`
- **Прокси отваливается** — Android усыпляет процессы. `install.sh` вызывает `termux-wake-lock` автоматически; при ручном запуске выполни его сам перед `tgproxy`
- **Ошибки при установке** — убедись, что Termux из F-Droid, а не Google Play
- **Высокий пинг** — попробуй другой DC в `config.json`
- **Telegram не подключается после рестарта** — порт и credentials изменились, обнови их в настройках прокси Telegram
